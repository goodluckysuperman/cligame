# PRD：cligame 第一版

## 1. 目标

构建一个运行在终端中的文字小说小游戏原型，让用户通过单个 CLI 命令进入故事，在阅读氛围化文本的同时做出少量关键选择，并走向不同结局。

## 2. 核心体验

- 启动命令简单，进入即玩。
- 故事篇幅短，首版控制在一次可完整游玩的范围内。
- 文本风格偏神谕感、宇宙恐怖、宗教颂词氛围。
- 用户在关键节点做 2-3 次选择，最终到达不同结局。
- 背景音效首版只提供接口与静音开关，不强依赖真实音频资源。

## 3. 目标用户

- 想快速体验短篇互动小说的玩家。
- 想基于现有语料继续扩写故事的创作者。
- 想在终端中尝试轻量叙事玩法的开发者。

## 4. 范围

### In Scope

- 一个可直接运行的 CLI 命令。
- 一个内置短篇故事。
- 故事文件的结构化加载能力。
- 轻分支、多结局。
- 基础终端排版。
- 测试、README、技术设计文档。

### Out of Scope

- 存档系统。
- 数值系统、背包、战斗。
- 长篇多章节内容管理。
- 真正跨平台的音频播放实现。
- 图形界面。

## 5. 功能需求

### 5.1 CLI

用户可以通过 `uv run cligame play` 进入游戏。

### 5.2 故事加载

系统支持：
- 读取内置故事。
- 从外部 JSON 文件读取故事。

### 5.3 交互流程

- 展示序章。
- 展示当前场景文本。
- 展示可选分支。
- 接收用户输入。
- 对非法输入重新提示。
- 到达结局后结束本轮游戏。

### 5.4 音频接口

- 默认提供背景音接口占位。
- 提供 `--mute` 开关。
- 不要求首版内置可播放音频资源。

## 6. 内容要求

- 首个故事必须可完整游玩。
- 至少包含 2-3 个不同结局。
- 风格参考仓库现有语料 `semantic-library.md`。
- 语料库作为写作素材，不直接作为流程脚本。

## 7. 验收标准

- `uv sync` 可成功完成环境同步。
- `uv run cligame play` 能启动并完成一轮游戏。
- `uv run cligame list-stories` 能列出可玩的内置故事。
- `uv run cligame play --mute` 可正常运行。
- 至少 2 个不同结局可手动走通。
- 非法输入不会导致程序崩溃。
- `uv run pytest` 和 `uv run ruff check .` 通过。
