# PRD：Threshold Protocol / 门限协议

## 1. 产品目标

把当前项目从轻分支文本原型升级为一款命令式终端收容游戏。玩家需要在有限时间内，通过终端命令调查事故、恢复或隔离模块、保存证据、判断信息真假，并最终执行一条收容协议。

## 2. 核心体验

- 开场任务明确：04:00 前阻止零号收容室异常同步。
- 玩家操作以英文命令为主，显示语言可保持中文叙述。
- 前辈作为高度可信的神秘导师，负责前期教学、任务发放和工具交接。
- 每个动作都会推动时间，迫使玩家取舍。
- 模块恢复不是纯收益，而是用风险换信息。
- 同一份文本可能发生变化，玩家必须依赖离线记录器保存证据。
- 玩家最后需要在多个协议间做决定，且结局主要代价偏向“牺牲自己”。

## 3. 第一夜玩法范围

### In Scope
- 倒计时系统
- 任务系统
- 模块状态系统
- 终端命令系统
- 离线记录器
- 前辈引导
- 动态文件变化
- 4 条终局协议

### Out of Scope
- 长期存档
- 多章节剧情
- 战斗系统
- 完整跨平台音频兼容
- 图形界面

## 4. 第一夜关键系统

### 4.1 倒计时
每个核心动作消耗时间，玩家无法看完所有内容。

### 4.2 模块系统
首版围绕五个模块：
- audio
- archive
- camera
- access
- auth

### 4.3 命令系统
首版重点命令：
- `help`
- `status`
- `modules`
- `ls`
- `open`
- `relay`
- `reply`
- `scan`
- `restore`
- `isolate`
- `record`
- `history`
- `whoami`
- `inventory`
- `protocols`
- `execute`

### 4.4 工具系统
首个关键工具是“离线记录器”，用于保存快照并识别文本篡改。

### 4.5 前辈角色
前辈负责：
- 解释事故
- 下发任务
- 教基础命令
- 发放工具
- 在中后段逐渐显露不寻常的信息掌握程度

## 5. 验收标准

- 玩家进入游戏后立即知道目标与剩余时间
- 前 5 分钟内完成基础命令教学
- 玩家能通过命令而不是数字选项推进流程
- 恢复模块会真实改变可读内容和玩家状态
- 至少存在一份会变化的文件，并能通过 `record` 识别问题
- 至少有 3 条可见协议，且至少 2 条不是默认开放
- `uv run pytest` 和 `uv run ruff check .` 通过
